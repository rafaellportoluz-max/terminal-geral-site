Arquitetura escalável

Adicionar um módulo é acrescentar um objeto

O núcleo nunca muda. Cada domínio da API é um handler independente em server.mjs; cada módulo do ecossistema é uma entrada em registry.json. Escalabilidade por composição, não por reescrita.

registry.json catálogo central 
// adicionar um agente = só acrescentar aqui. O núcleo não muda.
{
  "agents": [
    {
      "id": "claude-code",
      "nome": "Claude Code (CLI)",
      "funcao": "motor-ia",
      "kind": "cli",
      "apis": ["Anthropic"]
    }
  ],
  "projects": [ /* 14 projetos */ ],
  "integrations": [ /* Claude · Obsidian · MCPs · gh */ ],
  "automations": [ /* hooks · backup · launch */ ],
  "databases": [ /* JSON · SQLite · Postgres */ ]
}
01

Node.js puro

Só a biblioteca padrão: node:http, node:fs, node:os, child_process. Nenhum npm install.

02

Handlers independentes

Cada rota é isolada em ROTAS. Métricas, busca, agentes e integração Claude não se pisam.

03

Estado durável

data/ guarda estado, settings e logs JSONL append-only fora da conversa — memória que sobrevive à sessão.
